# 📘 Фаза 1. Фундамент .NET 10 и архитектура (Модули 1-6)

---

**Общая цель фазы:** Построить production-ready консольный стек на .NET 10: освоить современный синтаксис (top-level statements), `IHost` архитектуру, управление зависимостями, конвейер конфигурации, работу с опциями в реальном времени, обработку сигналов ОС и собрать эталонный pipeline приложения.

**Стек:** C# .NET 10, Microsoft.Extensions.Hosting, Microsoft.Extensions.Configuration, Microsoft.Extensions.DependencyInjection.

## 💡 Рекомендации преподавателя на эту фазу

1. **Host = Foundation.** Не пишите `while(true)` в `Main`. Используйте `BackgroundService` и `IHost`. Это стандарт для всего .NET.
2. **DI = Lifecycle Control.** Понимайте разницу между Singleton/Scoped/Transient. В консольных приложениях Scope нужно создавать явно через `IServiceScopeFactory`.
3. **Config = Priority Chain.** Помните порядок: CLI > ENV > JSON. Используйте `:` для иерархии в коде, `__` в переменных окружения.
4. **Options = Validation.** Используйте `IOptionsMonitor<T>` с `ValidateOnStart` для типизированного, валидированного конфига с hot-reload.
5. **Shutdown = Cooperation.** Обрабатывайте `CancellationToken` в циклах. Не игнорируйте сигналы завершения — это приводит к утечкам и повреждению данных.
6. **Baseline = Reproducible.** Parity-тесты, `dotnet publish`, структурированное логирование — это не опции, а требования к production-готовности.
7. **2026 Reality:** .NET 10 требует меньше бойлерплейта. Top-level statements + `IHost` + `BackgroundService` — это новый стандарт для консольных приложений. Zero tribal knowledge, zero magic.
