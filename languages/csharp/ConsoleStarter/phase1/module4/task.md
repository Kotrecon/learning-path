## Секция 3: Валидация конфигурации при старте

## Задача 4.3: Настроить валидацию конфига через DataAnnotations и ValidateOnStart

**Описание:** Добавить атрибуты [Required] и [Range] к свойствам AppSettings. Настроить валидацию через `AddOptions<T>().ValidateDataAnnotations().ValidateOnStart()`. Убедиться, что некорректный конфиг блокирует запуск с понятной ошибкой.

**Способы достижения:**

- Аннотировать свойства: `[Required] public string Name`, `[Range(1, 3600)] public int Timeout`
- Настроить валидацию: `builder.Services.AddOptions<AppSettings>().Bind(...).ValidateDataAnnotations().ValidateOnStart()`
- Удалить Name из JSON, запустить приложение, зафиксировать OptionsValidationException
- Закоммитить: `git commit -m "feat(module4): add configuration validation"`

**Результат:** Вывод исключения OptionsValidationException с указанием нарушенного правила, коммит с настройкой валидации в репозитории.
