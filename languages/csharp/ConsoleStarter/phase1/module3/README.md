# Модуль 3 — Configuration Pipeline

> Приоритет: **CLI > ENV > JSON**

## 🎯 Цель

Научиться управлять конфигурацией в .NET 10: понимать, что загружается автоматически, а что требует явного подключения.

---

## 🔑 Ключевые концепции: Явное vs Неявное

### 📦 Что .NET делает АВТОМАТИЧЕСКИ (неявно)

| Действие                                | Как работает                                                                 | Что нужно от тебя                                                                                                        |
| --------------------------------------- | ---------------------------------------------------------------------------- | ------------------------------------------------------------------------------------------------------------------------ |
| **Загрузка `appsettings.json`**         | `Host.CreateApplicationBuilder()` сам подключает `JsonConfigurationProvider` | ✅ Файл должен лежать в проекте с `<CopyToOutputDirectory>PreserveNewest</CopyToOutputDirectory>`                        |
| **Плоская структура ключей**            | Вложенный JSON `"App":{"Timeout":30}` → `"App:Timeout":"30"`                 | ✅ Использовать `:` как разделитель при чтении: `config["App:Timeout"]`                                                  |
| **Добавление провайдеров в цепочку**    | Порядок: JSON → ENV → CLI (последний побеждает)                              | ✅ Ничего, но помнить приоритеты при отладке                                                                             |
| **Чтение переменных окружения**         | `EnvironmentVariablesConfigurationProvider` грузит всё, что есть в процессе  | ✅ Использовать `__` вместо `:` в имени: `$env:App__Timeout="120"`                                                       |
| **Парсинг аргументов командной строки** | `CommandLineConfigurationProvider` парсит `--Key=Value`                      | ✅ Передавать `args` в билдер (уже есть по умолчанию) и использовать `--` при запуске: `dotnet run -- --App:Timeout=300` |

### 🔧 Что ты делаешь ЯВНО (требует кода)

| Действие                                  | Код / Конфиг                                                                                                                                                 | Зачем                                                            |
| ----------------------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------ | ---------------------------------------------------------------- |
| **Чтение значения из конфига**            | `builder.Configuration["App:Timeout"]`                                                                                                                       | `IConfiguration` не делает ничего сам, ты запрашиваешь ключ явно |
| **Типизированный доступ (`IOptions<T>`)** | 1. `record AppSettings { ... }`<br>2. `builder.Services.Configure<AppSettings>(...)`<br>3. `host.Services.GetRequiredService<IOptions<AppSettings>>().Value` | Биндинг не происходит «магически» — нужна регистрация + резолв   |
| **Приведение типов**                      | Автоматически при использовании `IOptions<T>`                                                                                                                | Без `IOptions<T>` все значения — `string`, парсинг вручную       |
| **Валидация конфига**                     | `.ValidateDataAnnotations()` при регистрации `Configure<T>`                                                                                                  | По умолчанию .NET не валидирует значения, только структуру       |
| **Hot-reload конфигурации**               | `reloadOnChange: true` + `IOptionsMonitor<T>`                                                                                                                | По умолчанию конфиг статичен, изменения в файле игнорируются     |
