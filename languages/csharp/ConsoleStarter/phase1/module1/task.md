# Модуль 1 — .NET 10 Console Modernization

> Цель: Создать базовый Program.cs с Host.CreateDefaultBuilder(), перейти на top-level statements, убрать boilerplate.

---

## Секция 1: Переход на top-level statements

### Задача 1.1: Мигрировать точку входа на top-level statements

**Описание:** Удалить класс Program и метод Main, заменив их на код корневого уровня. Понять, что компилятор автоматически генерирует точку входа. Убедиться, что проект компилируется и запускается с минимальным кодом. Инициализировать Git-репозиторий для контроля версий изменений.

**Способы достижения:**

- Создать проект: `dotnet new console -n ModernApp -lang C# -f net10.0`
- Удалить `class Program` и `static void Main`, оставить `Console.WriteLine("Hello .NET 10");`
- Инициализировать репозиторий: `git init`, `dotnet new gitignore`, выполнить первый коммит
- Проверить сборку: `dotnet build`, при необходимости проверить сгенерированный IL через SharpLab

**Результат:** Файл Program.cs с одной строкой кода, инициализированный Git-репозиторий с коммитом `feat(module1): migrate to top-level statements`, успешный вывод `dotnet build`.

## Секция 2: Инициализация IHost

### Задача 2.1: Настроить базовый хост через CreateDefaultBuilder

**Описание:** Подключить Microsoft.Extensions.Hosting и инициализировать хост через CreateDefaultBuilder. Понять, что хост управляет жизненным циклом, конфигурацией и логированием. Зафиксировать изменения в репозитории с понятным сообщением коммита.

**Способы достижения:**

- Добавить пакет: `dotnet add package Microsoft.Extensions.Hosting`
- В Program.cs добавить: `var host = Host.CreateDefaultBuilder(args).Build(); await host.RunAsync();`
- Запустить приложение, проверить завершение по Ctrl+C с кодом 0
- Зафиксировать: `git add . && git commit -m "feat(module1): implement basic IHost setup"`

**Результат:** Program.cs с инициализацией хоста, лог успешного запуска и завершения, коммит с настройкой IHost в истории репозитория.

- Сохранить лог в файл `logs/lifecycle-sequence.txt` для документации

**Результат:** Файл лога с последовательностью "Before RunAsync" → "Ctrl+C" → "After RunAsync", коммит с верификацией жизненного цикла в репозитории.
