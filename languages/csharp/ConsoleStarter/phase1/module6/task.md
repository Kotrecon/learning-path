**Результат:** Вывод консоли с логами формата `[INF] PipelineWorker: Worker started`, коммит с настройкой логирования в репозитории.

**Результат:** Лог `"Unhandled exception: ..."` при искусственной ошибке, код выхода `1`, коммит с обработкой ошибок в репозитории.

## Задача 6.5: Настроить публикацию self-contained билда для продакшена

**Описание:** Использовать `dotnet publish` с флагами `-c Release`, `-r <RID>`, `--self-contained true` для создания независимого билда. Проверить, что опубликованное приложение запускается без установленного .NET SDK.

**Способы достижения:**

- Выполнить: `dotnet publish -c Release -r win-x64 --self-contained true -o ./publish`.
- Проверить содержимое папки `./publish`: наличие `.exe` и всех зависимостей.
- Запустить `./publish/приложение.exe` в чистой среде, убедиться в работоспособности.
- Закоммитить: `git commit -am "chore(module6): add publish configuration for production"`.

**Результат:** Папка `./publish` с рабочим self-contained билдом, успешный запуск без .NET SDK, коммит с настройкой публикации в репозитории.
