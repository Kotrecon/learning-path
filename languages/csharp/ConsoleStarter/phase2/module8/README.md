# Модуль 8 — 📡 OpenTelemetry Integration

> Цель: Подключить OTel SDK, настроить консольный/OTLP экспорт для Traces, Metrics, Logs.

---

## 🚀 Быстрый старт

### Сборка и запуск (разработка)

```powershell
cd src/ConsoleStarter
dotnet run
```

**Ожидаемый вывод:**

- Трейсы (`Activity`) выводятся в консоль мгновенно при выполнении операций.
- Метрики (`Metric`) выводятся в консоль периодически (по умолчанию каждые 60 секунд).
- Приложение не падает, даже если OTel Collector недоступен на `localhost:4317`.

---

## ✅ Критерии приёмки

- [x] OTel SDK подключен через `AddOpenTelemetry()` с `OpenTelemetry.Extensions.Hosting`
- [x] `ActivitySource` создаёт трейсы с тегами (например, `ProcessData`)
- [x] `Meter` создаёт и инкрементирует счётчики (например, `items.processed`)
- [x] Console Exporter выводит Traces и Metrics в консоль
- [x] OTLP Exporter настроен на стандартный endpoint (`http://localhost:4317`, `Grpc`)
- [x] Все изменения зафиксированы в Git с понятной историей коммитов

---

## 🔑 Ключевые инсайты

| Концепция          | Инсайт                                                                                                                                |
| ------------------ | ------------------------------------------------------------------------------------------------------------------------------------- |
| **Инициализация**  | `TracerProvider` и `MeterProvider` ленивые. Они активируются только при `host.StartAsync()`, а не при `builder.Build()`.              |
| **ActivitySource** | Объявляется один раз (статически) на всё приложение. Спаны создаются в точках бизнес-логики, а не в `Program.cs`.                     |
| **Meter**          | Инструменты метрик (`Counter`, `Histogram`) потокобезопасны и создаются один раз. Метрики агрегируются и экспортируются периодически. |
| **OTLP Exporter**  | Работает в фоне. Если endpoint недоступен, данные молча теряются, приложение не падает.                                               |

## 🧪 Проверка вручную

1. **Штатный запуск:**

   ```powershell
   cd src/ConsoleStarter && dotnet run
   # Ожидаем: вывод Activity с тегами, затем через ~60 сек вывод Metric
   ```

2. **Проверка конфигурации OTLP:**
   Убедиться, что в `Program.cs` явно указаны:

   ```csharp
   options.Endpoint = new Uri("http://localhost:4317");
   options.Protocol = OpenTelemetry.Exporter.OtlpExportProtocol.Grpc;
   ```
