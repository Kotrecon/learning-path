## Секция 2: ActivitySource для трассировки

### [x] Задача 8.2: Реализовать создание трейсов через ActivitySource с тегами

- **Статус:** ✅ Completed
- **Дата выполнения:** 2026-06-10
- **Коммит:** `feat(module8.2): implement ActivitySource tracing with tags`
- **Заметки/Наблюдения:**
  - `ActivitySource` — фабрика, объявляется один раз (статически) на всё приложение.
  - `Activity` (спаны) создаются в местах бизнес-логики, а не в `Program.cs`.
  - `TracerProvider` ленивый: инициализируется при `host.StartAsync()`, а не при `builder.Build()`.
  - Для демонстрации в `Program.cs` потребовалось заменить `host.RunAsync()` на `host.StartAsync()` + `WaitForShutdownAsync()`.
  - 💡 Инсайт: Трейсы — это не "один спан на приложение", а множество спанов в ключевых точках бизнес-операций.

## Секция 3: Meter для кастомных метрик

### [x] Задача 8.3: Создать инструменты метрик через Meter API

- **Статус:** ✅ Completed
- **Дата выполнения:** 2026-06-10
- **Коммит:** `feat(module8.3): implement Meter for custom metrics`
- **Заметки/Наблюдения:**
  - `Meter` — фабрика для инструментов метрик (Counter, Histogram, Gauge).
  - В top-level statements нельзя использовать `static readonly`, только `var`.
  - Метрики экспортируются по умолчанию каждые 60 секунд (периодический экспорт).
  - Для более частого экспорта можно настроить `MetricReaderOptions.ExportIntervalMilliseconds`.
  - 💡 Инсайт: Метрики — это не мгновенный вывод, а агрегированные данные за период.

## Секция 4: Интеграция экспортеров Console и OTLP

### [x] Задача 8.4: Настроить вывод сигналов в консоль и через OTLP-протокол

- **Статус:** ✅ Completed
- **Дата выполнения:** 2026-06-10
- **Коммит:** `feat(module8.4): integrate Console and OTLP exporters for Traces and Metrics`
- **Заметки/Наблюдения:**
  - `AddConsoleExporter` используется для локальной отладки (мгновенный вывод).
  - `AddOtlpExporter` настроен для продакшена.
  - Приложение не падает, если OTel Collector недоступен (экспортер молча теряет данные, что является ожидаемым поведением для фоновой отправки).
  - 💡 Инсайт: Экспортеры могут работать параллельно, дублируя поток сигналов для разных целей (dev vs prod).
