4. `InMemoryExporter` используется для тестов.
5. `module12/README.md` описывает конфигурацию экспортеров.

## 💡 Рекомендации преподавателя на эту фазу

1. **Structured Logs = Data, not Text.** Используйте JSON Console Formatter. Message templates должны содержать параметры `{Name}`, не интерполяцию `$"{Name}"`.
2. **OTel = Unified Protocol.** Traces, Metrics, Logs собираются одним SDK. Не используйте зоопарк агентов.
3. **Correlation = Glue.** `TraceId` связывает логи и трейсы. `Baggage` пробрасывает бизнес-контекст. Без этого observability слепа.
4. **Metrics = Business Signals.** Считайте ошибки, замеряйте длительность, мониторьте очередь. Это данные для принятия решений.
5. **Enrichment & Filtering = Signal/Noise.** Обогащайте логи контекстом (User, Env), фильтруйте шум сторонних библиотек.
6. **Exporters = Reliable.** Используйте batching и retry. InMemory exporter — для тестов, Console — для дебага, OTLP — для продакшена.
7. **Pipeline = Testable.** Пишите parity тесты с `InMemoryExporter`. Валидируйте схему данных. Sampling — для управления нагрузкой.
8. **2026 Reality:** .NET 10 имеет встроенную поддержку OTel. `JsonConsoleFormatter` — стандарт. `InMemoryExporter` — основа тестирования observability. Zero magic, pure data.
